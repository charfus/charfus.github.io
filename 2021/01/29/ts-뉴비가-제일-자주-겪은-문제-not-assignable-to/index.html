TS :: null 혹은 undefined 피하기

2021-01-29
2021-01-29
javascript, operator, typescript

얼떨결에 무작정 타입스크립트로 간단한 작업을 몇 번 하게 되었는데 아직 익힌지 얼마 안 되어서 코드 작성하는 내내 그냥 얼레벌레인 상태다. 공식 Doc에 나오는 Basic Usage에서 쌓아가는, 그런 갈길이 먼 상태… 아무튼 그런 상태에서도 계속해서 부딪히는 문제가 있어서 우선 기록은 남겨놓으려고 한다.

아직 정확한 용어를 숙지하지 못해서 그냥 풀어서 쓰면, undefined 나 null를 포함하여 복수의 타입이 허용되는 변수를 사용할 때 내가 특정한 타입, 예를 들어 string이나 number 같은 타입을 할당하려는 데에도 불구하고 undefinde이나 null도 ‘가능’하기 때문에 “Type … is not assignable to …”와 같은 오류와 빨간 줄이 뜨는 상태에 자주 처했다.

아마 이 문제 때문에 구글링을 하면 가장 먼저 뜨는 스택오버플로우 질문이 아닐까 싶은데

Stackoverflow :: Typescript: Type ‘string | undefined’ is not assignable to type ‘string’

이걸 읽고 급하게 대처하긴 했는데 제대로 읽진 않아서 다시 확인해보았다.

Non-null assertion operator - Non-null 단언 연산자 ( ! )

가장 유용하다고 선택된 답변은 non-null assertion operator를 써서 null이나 undefined가 아니라는 걸 타입스크립트에게 알려주면 된다는 것이었다. 역시나 급하게 대처해야하는 상황에 적절한 방법이였다!

공식 문서 TS 2.0 - Non-null assertion operator

“x!는 null과 undefined를 제외한 타입 x의 값을 뱉어내고, x 와 x as T 같은 타입 단언과 마찬가지로 자바스크립트 코드에서는 제거된다”고 한다. 그래서 위 스택오버플로우 질문에 이런 답변도 있던 것 같다.

어쩐지 어떤 개발자들은 분명 싫어할 주먹구구식 방법 같다는 인상을 강하게 받아 계속해서 살펴보니 이런저런 반대 의견들이 있었다. 실제 런타임에서 값으로 null이나 undefined를 갖고 있으면 어쨌든 오류가 발생하리라는 점이 지적되는 것 같다. 차라리

이렇게 직접 검사를 하라는 조언도 있었다. 출처 - sonarsource 위의 스택오버플로우의 답변에 대한 댓글에도 이와 비슷한 조언이 있었다.

Nullish coalescing operator - Null 병합 연산자 ( ?? )

또 다른 답변으로는 ||와 비슷한 Null 병합 연산자를 쓰라는 조언이 있었다.

person.name이 null 혹은 undefined 일 때, 오른쪽 ‘’을 반환하고 그렇지 않으면 왼쪽 피연산자를 반환한다는 뜻이라고 한다. MDN을 참고하면 ||와 다르게 falsy 값 (0, ‘’, NaN 등) 이어도 null과 undefined만 아니면 왼쪽 피연산자를 반환한다. (||)은 falsy 값이면 무조건 오른쪽 디폴트 값을 반환한다. 이 연산자는 TS 3.7부터 도입되었다고 한다.

자바스크립트에 이런 연산자들이 있다고 어렴풋이 들긴 했는데 타입스크립트와 결합되어서 제 역할을 한다는 인상을 받았다. 사실 직접 코드를 작성할 당시 느꼈던 점은 이러한 연산자들을 써야할 때 애초에 내가 타입을 정의하고 설계한 시점부터 에러의 씨앗을 심어졌다는 것이였는데 (…) 정말 어쩔 수 없는 경우에만 저런 연산자들을 쓸 수 있도록 코드 좀 잘 짜보고 싶다.
